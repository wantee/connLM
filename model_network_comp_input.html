The Input Layer

Introduction

The input layer specify how we choose words in a sentence and use them as the input to the network.

Topology Configurations

The input tag have following properties:

  • context – whose value should be a list of integers separated by commas. This values denote the relative position with target word, negative values are on the left (history), whilst the positives are on the right (future). E.g. for a 5-gram input, context=-4,-3,-2,-1, or one can also set a skip-gram input with context=-4,-2. Since our model should be able to predict current target word, 0 can not be in the context. In addition, one could give each position in context a specific weight or scale, by appending a colon and a float number to the corresponding integer. E.g., context=-4:0.5,-3:0.8,-2:0.9,-1. If a weight is not given, the default value 1.0 will be assigned.
  • combine – this specify how will the context words be combined into a single vector as input for the model. There are three methods: sum, avg and concat, which means take the summation of all context words, take the average value of all context words and concatenate the context words to a longer vector.

NOTE that, when input layer is followed by a direct_glue, the meaning of these properties will be different. The context is to specify the n-gram features fed into Maximum Entropy model and combine would be invalidated. Therefore, when context=-4,-3,-2,-1, it actually means we will use 4 n-gram features as input. This is bit of confusing, hopefully this would be changed until we figure out a better way to specify the n-gram feature for Maximum Entropy model.
