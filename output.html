The Output Layer

Introduction

A connLM model can only contain one output layer, which is shared by one or more component models.

An output layer must be generated before other components added.

Usage

The related source code is in output.h. output_generate() can be called to generate a output layer with specific options. During training, output_loss() is used to get objective loss for output layer. And output_get_prob() can be used to get word prob after forward pass.

The command line interface for generating output layer is connlm-output, available options are:

$ connlm-output --help
Usage : connlm-output [options] <model-in> <model-out>
Options :
--help : Print help (bool, default = false)
--log-file : Log file (string, default = "/dev/stderr")
--log-level : Log level (1-8) (int, default = 8)
--class-size : Size of class layer (int, default = 100)
--hs : Hierarchical softmax (bool, default = false)
--max-code-len : Maximum length for code used by HS (int, default = 40)
--class-hs : Whether using HS for classes (bool, default = false)
--binary : Save file as binary format (bool, default = true)
--config : config file (string, default = "")

ConnLM supports three kind of output layer: 1-of-V vector, class based factorization and hierarchical softmax.

–class-size can be used to set the number of classes used in class based factorization. If it is less or equal to zero, no class based factorization will be applied.

–hs option determines whether to use hierarchical softmax. –max-code-len and –class-hs give more fine control of HS. –max-code-len is the maximum length of code used to build HS tree, i.e., the maximum depth of the tree. The reason of such a option is for convenient memory allocation, if it complains that max-code-len is too small, one should enlarge this value. If –class-hs is set to be true and –class-size is bigger than zero, it will also apply HS on the class output layer.

The class based factorization and HS can be applied in one output layer simultaneously. And if both of them are not used, connlm-output will generate a 1-of-V vector output layer.

Note that, there are some constraints on the using of HS:

  1. HS is not supported by MaxEnt model.
  2. with HS, the pre-output layer of all component models must be the same size.
